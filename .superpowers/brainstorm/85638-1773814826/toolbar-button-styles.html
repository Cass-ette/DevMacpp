工具栏按钮样式

基于方案 A 的完整布局，选择按钮的视觉风格

A

经典图标按钮

图标 + 文字标签，和原版 Dev-C++ 5.11 最接近

优点

  • 最接近原版风格
  • 图标+文字，功能清晰

缺点

  • 占用垂直空间较多
B

纯图标按钮

只显示图标，鼠标悬停显示提示，更紧凑

优点

  • 紧凑，节省空间
  • 编辑区更大

缺点

  • 需要记住图标含义
  • 和原版差异较大
C

图标+文字横排

图标在左，文字在右，横向排列，兼顾清晰和紧凑

优点

  • 功能清晰，易识别
  • 比方案 A 更紧凑
  • 现代感更强

缺点

  • 和原版布局略有不同
D

macOS 原生风格

使用 macOS 系统风格的按钮，SF Symbols 图标，更符合 Mac 应用习惯

优点

  • 最符合 macOS 应用风格
  • 视觉精致，现代
  • 用户体验好

缺点

  • 和原版 Dev-C++ 差异最大
  • 可能"太 Mac"了

说明：这里用 emoji 做演示图标，实际开发会用真正的图标资源（SF Symbols 或自定义图标）。重点是选择按钮的布局和风格。
